# ESPHome 智能插座开发实录：ESP32-MINI-1 + BL0940 + LD2412 避坑指南与最终源码

## 前言
在智能家居 DIY 领域，86 型墙壁智能插座一直是个热门项目。本次量产固件基于 **ESP32-MINI-1** 主控，集成了 **BL0940 电量计量芯片**、**LD2412 毫米波人体存在雷达**、磁保持继电器以及红外/射频预留接口。

在将各个模块接入 ESPHome 并对接 Home Assistant 的过程中，我经历了无数次编译报错、实体冲突和串口通信失败。本文将毫无保留地分享这些“踩坑”经验，并提供一份经过千锤百炼的最终稳定代码。

---

## 🕳️ 核心踩坑记录（干货预警）

### 坑 1：UART 串口并联与波特率冲突
**现象**：最初误以为 BL0940 和 LD2412 可以接在同一对引脚上。
**真相**：UART 是点对点通信，绝对不能并联！BL0940 波特率为 4800，LD2412 为 115200（或 256000）。ESP32 的一个 UART 控制器只能设置一个波特率，并联会导致双方数据全部乱码。
**解决**：必须为两个芯片分配独立的物理 UART 引脚。

### 坑 2：ESP32-MINI-1 的 GPIO18/19 引脚陷阱
**现象**：使用 GPIO18/19 连接 LD2412，代码配置无误，但雷达状态永远显示“未知”。
**真相**：ESP32-MINI-1 的 GPIO18 和 GPIO19 是 SPI Flash/PSRAM 的复用引脚。在 `esp-idf` 框架下，系统底层会默认尝试占用这两个引脚，导致软件串口通信彻底失败。
**解决**：
1. 在 `esp32:` 节点下增加 `sdkconfig_options`，强制禁用 PSRAM 并锁定 Flash 模式。
2. 增加 `rx_buffer_size: 2048` 防止高波特率丢包。
*(注：为了追求极致稳定，本项目最终妥协使用了 ESP32 的原生 UART0 引脚 **GPIO1/GPIO3**，并降级波特率至 115200。)*

### 坑 3：中文命名引发的“实体 ID 冲突”惨案
**现象**：使用纯中文名称（如 `LD2412 有人`、`当前有人时长`），编译时报错 `Duplicate entity`。
**真相**：ESPHome 在生成底层 C++ 变量名和 HA 实体 ID 时，**会无情地过滤掉所有中文字符**。导致多个中文实体变成了相同的空白 ID。
**解决**：在中文名称后追加唯一的英文后缀。
**⚠️ 致命细节**：**绝对不能用括号 `()`！** 例如 `当前有人时长 (Curr)`，ESPHome 同样会过滤掉括号，导致和 `当前无人时长 (Curr)` 再次冲突。
**正确做法**：使用空格加下划线，如 `当前有人时长 P_Curr`。

### 坑 4：`object_id` 属性不支持
**现象**：试图通过添加 `object_id: ld2412_target` 来解决中文冲突，结果报错 `invalid option`。
**真相**：`switch.ld2412` 等部分组件在底层代码中硬编码了不支持 `object_id` 属性。
**解决**：放弃 `object_id`，老老实实修改 `name` 加英文后缀。

### 坑 5：ESPHome 的 OTA 升级误区
**现象**：设备进入 AP 热点模式后，试图在 `192.168.4.1` 网页中上传 `.bin` 文件进行升级，找不到入口。
**真相**：ESPHome 原生**不支持**网页端直接上传固件（那是 Tasmota 的做法）。`192.168.4.1` 仅用于配网。
**解决**：
1. 首次烧录必须使用 USB 线刷 `.factory.bin`。
2. 后续 OTA 升级必须通过 ESPHome Dashboard 点击 `Install -> Wireless (OTA)` 自动推送 `.ota.bin`。

    name: "button"

    resistor_two: 1950

    lambda: |-

        id(var_energy) = id(bl0940_energy).state;

      return id(bl0940_energy).state - id(var_energy);

  - platform: ld2412
    moving_distance: { name: "LD2412 运动距离 MDist", id: ld2412_moving_dist }
    still_distance: { name: "LD2412 静止距离 SDist", id: ld2412_still_dist }

    detection_distance: { name: "LD2412 检测距离 Det", id: ld2412_det_dist }
    light: { name: "LD2412 光照度 Lux", id: ld2412_light }

  # 时长统计传感器（注意：名称中无括号，使用空格+英文）
  - platform: template
    name: "当前有人时长 P_Curr"
    id: sensor_curr_presence
    unit_of_measurement: "s"
    accuracy_decimals: 0
    lambda: return id(current_presence_seconds);
    update_interval: 1s

  - platform: template
    name: "上次有人时长 P_Last"
    id: sensor_last_presence
    unit_of_measurement: "s"
    accuracy_decimals: 0
    lambda: return id(last_presence_seconds);
    update_interval: 5s

  - platform: template
    name: "当前无人时长 A_Curr"
    id: sensor_curr_absence
    unit_of_measurement: "s"
    accuracy_decimals: 0
    lambda: return id(current_absence_seconds);
    update_interval: 1s

  - platform: template
    name: "上次无人时长 A_Last"
    id: sensor_last_absence
    unit_of_measurement: "s"
    accuracy_decimals: 0
    lambda: return id(last_absence_seconds);
    update_interval: 5s

  - platform: wifi_signal
    name: "WiFi RSSI"
    id: sensor_wifi_rssi
    update_interval: 60s

ld2412:
  uart_id: uart_ld2412

number:
  - platform: bl0940
    current_calibration: { name: Current Calibration }
    voltage_calibration: { name: Voltage Calibration }
    power_calibration: { name: Power Calibration }
    energy_calibration: { name: Energy Calibration }
  - platform: ld2412

    max_distance_gate: { name: "LD2412 最大距离门 MaxGate", id: ld2412_max_gate }
    light_threshold: { name: "LD2412 光照阈值 Thresh", id: ld2412_light_thresh }

time:
  - platform: homeassistant
    id: homeassistant_time
    on_time:
      - seconds: 0
        minutes: 0
        hours: 0
        then:
          - lambda: id(var_energy) = id(bl0940_energy).state;

button:
  - platform: bl0940
    name: Reset Calibration
  - platform: factory_reset
    name: Factory Reset
    internal: True
    id: btn_factory_reset
  - platform: ld2412
    factory_reset: { name: "LD2412 恢复出厂 Reset", id: ld2412_factory_reset }
    restart: { name: "LD2412 重启雷达 Restart", id: ld2412_restart }
    query_params: { name: "LD2412 查询参数 Query", id: ld2412_query }

text_sensor:
  - platform: ld2412
    version: { name: "LD2412 固件版本 Ver", id: ld2412_version }

    name: "ESPHome Version"

# 核心逻辑：每秒累加时长，每5分钟检查雷达是否离线并自动重启
interval:
  - interval: 1s
    then:
      - if:
          condition: { binary_sensor.is_on: ld2412_target }
          then:
            - lambda: |-

            - lambda: |-

                id(current_presence_seconds) = 0;

            lambda: return isnan(id(ld2412_det_dist).state);

## 🤖 Home Assistant 自动化实战建议

得益于代码中增加的“时长统计”功能，我们在 HA 中可以实现极其精准的自动化，彻底告别传统红外人体传感器的“关灯”痛点：

1. **人离开房间 10 分钟后自动关灯/关空调**
   * **触发器**：`上次无人时长 A_Last` > 600 (秒)
   * **优势**：比传统的“无人持续 10 分钟”更可靠，因为它记录的是“上一次无人状态的完整时长”，不会因为雷达偶尔的误判而重置计时器。

2. **人在房间久坐超过 1 小时，自动调暗灯光/提醒休息**
   * **触发器**：`当前有人时长 P_Curr` > 3600 (秒)

3. **雷达死机自愈监控**
   * 代码中已内置看门狗，每 5 分钟检测一次 `LD2412 检测距离 Det`。如果变成 `NaN`（未知），设备会自动执行软重启，实现真正的无人值守。

## 🤖 Home Assistant 自动化实战建议

得益于代码中增加的“时长统计”功能，我们在 HA 中可以实现极其精准的自动化，彻底告别传统红外人体传感器的“关灯”痛点：

1. **人离开房间 10 分钟后自动关灯/关空调**
   * **触发器**：`上次无人时长 A_Last` > 600 (秒)
   * **优势**：比传统的“无人持续 10 分钟”更可靠，因为它记录的是“上一次无人状态的完整时长”，不会因为雷达偶尔的误判而重置计时器。

2. **人在房间久坐超过 1 小时，自动调暗灯光/提醒休息**
   * **触发器**：`当前有人时长 P_Curr` > 3600 (秒)

3. **雷达死机自愈监控**
   * 代码中已内置看门狗，每 5 分钟检测一次 `LD2412 检测距离 Det`。如果变成 `NaN`（未知），设备会自动执行软重启，实现真正的无人值守。
